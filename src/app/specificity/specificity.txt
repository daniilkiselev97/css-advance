CSS Specificity (специфичность CSS) — это механизм, который определяет, какие правила CSS применяются к элементу, когда несколько правил могут потенциально применять стили к одному и тому же элементу. Специфичность играет ключевую роль в каскадировании стилей и позволяет разрешать конфликты между правилами, обеспечивая, что наиболее специфичные правила имеют приоритет.

Как работает специфичность
Каждое правило CSS имеет свой уровень специфичности, который определяется количеством и типами селекторов, используемых в правиле. Специфичность вычисляется на основе следующих критериев:

1.Inline Styles (Инлайновые стили): Наивысшая специфичность. Применяются непосредственно к элементу в атрибуте style, например:
<div style="color: red;">Text</div>

2.ID Selectors (Селекторы по ID): Высокий уровень специфичности. Пример:
#myId {
  color: blue;
}

3.Class, Attribute, and Pseudo-class Selectors (Селекторы по классу, атрибуту и псевдоклассы): Средний уровень специфичности. Примеры:
.myClass {
  color: green;
}
[type="text"] {
  color: orange;
}
:hover {
  color: purple;
}

4.Element and Pseudo-element Selectors (Селекторы по элементам и псевдоэлементам): Низкий уровень специфичности. Примеры:
div {
  color: black;
}
::before {
  content: "";
}

Конфликт между классом и элементом
div {
  color: black;
} - 0-0-0-1(А-B-C-D)

.myClass {
  color: red;
} - 0-0-1-0(А-B-C-D)

Приоритетность: При конфликте между правилами с разной специфичностью, правило с более высокой специфичностью будет иметь приоритет и применяться к элементу.
Каскадирование: Если два правила имеют одинаковую специфичность, то применяется правило, которое идет последним в CSS (в порядке их объявления).
Переопределение: Правила с более высокой специфичностью могут переопределять правила с меньшей специфичностью, даже если последняя находится после первой в CSS.
